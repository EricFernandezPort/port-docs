
- [`groups`](https://docs.gitlab.com/ee/api/groups.html) - Groups are how GitLab organizes repositories. Groups can also contain subgroups.
- [`projects`](https://docs.gitlab.com/ee/api/projects.html) - Projects in GitLab represent repositories. They hold source code, issues, merge requests and more.
- [`issues`](https://docs.gitlab.com/ee/api/issues.html) - Issues are used to track ideas, enhancements, tasks, or bugs in your projects.
- [`merge-requests`](https://docs.gitlab.com/ee/api/merge_requests.html) - Merge requests (MRs) are how you propose changes to your code stored in GitLab.
